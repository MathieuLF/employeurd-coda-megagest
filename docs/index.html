EmployeurD-MegaGest

Convertir une écriture détaillée EmployeurD en fichier MND MégaGest, avec des contrôles clairs avant de créer le fichier final.

Séquence guidée

Une conversion qui reste lisible du début à la fin.

Pas besoin de manipuler des commandes. L'application accompagne le choix des fichiers, la validation, puis la création du MND dans un dossier horodaté.

1

Ajouter l'écriture

Sélectionner l'écriture détaillée standard de EmployeurD, au format TXT.

2

Confirmer les soldes

Joindre le SPD640-P au besoin pour corroborer les totaux débit et crédit.

3

Créer le MND

Produire le fichier seulement lorsque les validations obligatoires sont conformes.

Aperçu

Tout ce qui compte tient dans le même écran.

L'utilisateur voit l'écriture EmployeurD, le rapport SPD640-P, les choix de sortie, le résumé et le journal sans chercher dans des fichiers techniques.

Contrôles intégrés

Le fichier final n'est pas créé si un écart bloquant apparaît.

Le but n'est pas seulement de convertir. C'est de produire un fichier que l'on peut vérifier calmement avant de le tester dans MégaGest.

Écriture EmployeurD

Longueur fixe, date, montant, lot, comptes et équilibre débit/crédit.

MND relu

Le MND généré est relu et ses propres totaux sont comparés au fichier source.

SPD640-P

Lorsque le rapport est fourni, ses totaux peuvent confirmer les soldes de la paie.

Données sensibles

Les fichiers de paie restent sur le poste.

L'application ne téléverse pas les fichiers EmployeurD, SPD640-P, MND, Markdown ou JSON. La vérification de mise à jour consulte GitHub sans transmettre de données de paie.

Les exemples publiés dans le dépôt sont synthétiques. Aucun fichier réel ne doit être joint à un billet public.
